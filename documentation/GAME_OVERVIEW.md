# Game Overview

## Project Summary

Un-Dead Hotel is a web-based, real-time colony/survival management game set in an infinitely generated hotel environment called **Hilbert's**.

The game mixes:

- world exploration,
- survivor management,
- scavenging and crafting,
- long-horizon base survival.

The core fantasy is "Fallout Shelter + RimWorld in an infinite post-apocalyptic hotel." The Hilbert's paradox framing is world flavor, not a separate puzzle system. The only paradox-driven mechanic is the infinite nature of the hotel.

## Game Pillars

- **Stay alive under scarcity:** food, water, and power pressure drive decisions.
- **Run a survivor colony in real time:** the player issues direct orders and handles priorities under threat.
- **Expand through dangerous unknown space:** every expansion move increases access to resources and risk.
- **Defend what matters:** zombies are basic but relentless, so layout and survivor positioning matter.

## Visual Direction

- Top-down perspective.
- Simplistic, readable visuals over high detail.
- Clear room silhouettes and threat readability.
- Tone: improvised post-apocalyptic survival inside a decaying hotel.

## Current Build Status

The codebase currently provides two runtime routes:

- `index.html`: game runtime
- `debug.html`: procedural generation debug mode

The procedural generation system is deterministic by chunk coordinate and world seed and is shared by all runtime routes.

Current game runtime gameplay slice includes:

- multi-human roster with one player `survivor` plus naturally spawned `guests`,
- survivor/guest visual differentiation and survivor-only direct selection policy,
- guest natural population policy anchored to living survivors (`1:10` static ratio from zombie target count),
- guest vision/perception loop (cone + line-of-sight) with wander/flee behavior,
- survivor-touch conversion (guest becomes survivor and immediately becomes selectable),
- left-click/drag-box survivor selection with `Shift + Left Click` toggle support,
- `Ctrl + Left Click` group move command for selected survivors using sub-tile bidirectional A* with:
  - 8-direction pathing (cardinal + diagonal),
  - corner-cut prevention for diagonal movement,
  - boundary-aware directional retry expansion,
  - unique quarter-tile destination claiming per survivor,
- locomotion standardization (Finding Our Way Phase 1):
  - survivors, guests, and zombies all execute movement on the same `0.25` tile grid,
  - zombie wander/pursuit and guest wander/flee now execute rasterized path arrays,
  - vector waypoint steering execution is removed from active gameplay movement,
- first-contact zombie population policy:
  - startup target of `100` zombies,
  - spawn band constrained to `10-100` tiles from the first human,
  - perimeter recycle (out-of-range zombies are despawned and replaced),
- zombie perception/combat loop:
  - cone + line-of-sight target acquisition,
  - nearest-target pursuit with last-known investigate fallback,
  - pursuit movement updates through short-cadence raster path replanning,
  - touch attacks for `20` HP per hit on `1.0s` cooldown,
  - zombie speed fixed at `50%` of human speed,
- always-visible HP bars for human and zombies with zombie cooldown bars,
- game-over overlay when no humans remain (simulation continues),
- backquote-toggle debug mode for path/collider/AI diagnostics (human + zombie overlays and text diagnostics),
- zoom pipeline guardrails that keep zoom feel intact while reducing high-zoom render lag (documented in `GAME_RUNTIME.md`).

## Core Setting

- Hilbert's is infinite and procedurally generated by chunk.
- Hotel amenities can repeat endlessly (multiple gyms, restaurants, kitchens, terraces, rooftops).
- There is no absolute top or bottom of the hotel.
- Stairwells create a lore-consistent route for endless inbound threats (and later, potential survivors).
- "Rooftops" on a floor are modeled as large terrace/balcony sections on that same floor.

## Core Gameplay Loop

1. Explore new rooms and corridors under fog of war.
2. Scavenge food, tools, and survival equipment.
3. Recruit and direct survivors.
4. Craft weapons and defensive capability.
5. Stabilize food/water/power access.
6. Expand colony footprint while managing risk.

## System Direction

### Resources and Infrastructure

- Early survival relies on scavenged food from kitchens and minibars.
- Hotel utilities (water/electricity) are expected to degrade and eventually fail as part of the apocalypse simulation.
- Recovery paths include rainwater collection on terraces/rooftops and salvaged solar power banks.
- If scavenged resources run low, survivors can transition to bootleg hydroponics/window farming and limited rooftop bird hunting.
- Short-term player objective: survive by securing reliable food first.

### Survivors

- Survivors can be discovered in locked hotel rooms and later can also arrive via stairwells.
- MVP behavior: survivors are simple worker units under full player control.
- Survivors can craft their own weapons in the MVP loop.
- Post-MVP direction: reduce room-spawn plausibility over time (declining probability / time-gated spawning), add autonomy, then add personality variation.

### Zombies

- Zombie pressure is effectively infinite due to stairwell access.
- MVP zombie behavior is intentionally simple:
  - wander aimlessly,
  - detect survivors by sight,
  - pursue nearest visible survivor,
  - attack on touch cooldown.
- Zombie style target: slow, dumb, "Walking Dead"-like shamblers.

### Vision, Territory, and Control

- The game is strictly real-time (not turn-based).
- Fog of war is strict: unexplored or currently unobserved space is hidden.
- Visibility should be driven by survivor vision cones.
- There is no explicit room-claim/base-boundary mechanic.
- Base footprint is defined by survivor presence and practical control, not ownership tags.

### End-State Philosophy

- No fixed endgame victory condition.
- The experience is an open-ended survival management sandbox.
- The practical failure state is colony collapse (loss of survivable workforce/capability).

## Gameplay Direction

### MVP Horizon

- Single-floor infinite world generation.
- Real-time survivor control with simple worker behavior.
- Strict fog of war + survivor vision cone reveal.
- Basic zombie threat loop (slow shambler AI).
- Scavenge food first, then craft and defend.
- Survivors primarily discovered in rooms.
- No room-claim/base-border system.

### Post-MVP Horizon

- Vertical expansion (multi-floor traversal).
- Progressive utility shutdown and long-run infrastructure collapse.
- Renewable and improvised sustainability systems (rain capture, solar banks, hydroponics/farming).
- Survivor recruitment model refinement (time-based plausibility and stairwell arrivals).
- Survivor behavior progression from direct control toward autonomy and personalities.
